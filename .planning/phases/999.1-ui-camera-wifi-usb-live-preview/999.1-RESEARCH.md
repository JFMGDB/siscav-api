# Phase 999.1 — Research: câmara USB / Wi‑Fi + preview em Next.js (TypeScript)

**Date:** 2026-04-05  
**Question:** O que é preciso saber para planear bem a fase (cliente separado + API SISCAV)?

## Summary

- **USB / browser:** `navigator.mediaDevices.getUserMedia({ video: true })` em **Client Component** (`'use client'`). Next.js App Router: componentes que tocam em DOM/API do browser não podem ser Server Components. Listar dispositivos: `enumerateDevices()` após permissão (ou `deviceId` no constraint).
- **Preview local:** `<video ref={} autoPlay playsInline muted />` + `ref.current.srcObject = stream`. Cleanup: `stream.getTracks().forEach(t => t.stop())` no unmount ou ao trocar fonte.
- **HTTPS:** getUserMedia exige contexto seguro (HTTPS ou `localhost`). Documentar para operadores em produção.
- **Wi‑Fi / IP:** Três padrões comuns:
  1. **MJPEG** — muitas câmaras expõem `multipart/x-mixed-replace`; em browsers costuma funcionar com `<img src={url} />` (atualização contínua), não `<video>`.
  2. **HLS** — `.m3u8`; Safari nativo; Chrome/Firefox frequentemente precisam **hls.js** num `<video>` ou canvas.
  3. **Snapshot polling** — `fetch` periódico de JPEG + `URL.createObjectURL` ou data URL (mais CPU/rede; simples).
- **CORS no stream:** Se a URL da câmara é noutro host sem CORS para `fetch`, `<img>` para MJPEG ainda pode funcionar (no-cors visual). Se precisar `fetch`, pode falhar — alinhado com CONTEXT: **sem proxy obrigatório na API** no MVP; operador na mesma LAN/VPN.
- **SISCAV API:** JWT já documentado em `docs/api/FRONTEND_INTEGRATION.md`. Preview **não** precisa de token; login do operador sim. CORS em `main.py` já inclui `http://localhost:3000` (Next default).
- **Env no Next:** `NEXT_PUBLIC_SISCAV_API_URL` (ex. `http://localhost:8000`) para chamadas à API; não misturar com URL da câmara IP.

## Risks / pitfalls

- **Strict Mode** React 18: double mount em dev pode pedir permissão duas vezes — tratar cleanup.
- **Safari iOS:** limitações a `getUserMedia` e autoplay; fase assume **desktop web** primeiro (per CONTEXT).
- **Mixed content:** página HTTPS não carrega stream HTTP da câmara — documentar.
