## objective

Protect `POST /api/v1/access_logs/` with a documented device ingest credential: header **`X-Device-Key`** compared to **`DEVICE_INGEST_KEY`** using **`secrets.compare_digest`**, registered in OpenAPI via **`APIKeyHeader`**, without changing the multipart body (`file` + `plate`).

**Purpose:** Close unauthenticated public ingest (SEC-01) while keeping integrator contracts visible in OpenAPI.

**Output:** Runtime enforcement, env documentation, and integration tests that send the header in CI.

## must_haves

1. `POST /api/v1/access_logs/` returns **401** when the key is missing, empty, or wrong (generic `detail`, no hint which check failed).
2. When **`DEVICE_INGEST_KEY`** is set, a request with matching **`X-Device-Key`** succeeds as today (200 + log payload) for valid multipart input.
3. OpenAPI documents **`X-Device-Key`** as an API key security requirement for that route (generated schema shows the header name).
4. **`env.local.example`** documents **`DEVICE_INGEST_KEY`**, **`ENVIRONMENT`**, and the dev-only bypass rule per **D-03** (development/unset + unset key may skip validation).
