## objective

Close **LOG-03** and **D-03**: documentation and OpenAPI **honestly** state that **stored images** are available only under policy — for v1, **administrator-only** (`get_current_admin_user`) on `GET /api/v1/access_logs/images/{image_filename}`, while **metadata list** remains any authenticated user (`get_current_user` on `GET /api/v1/access_logs/`).

**Purpose:** Prevent operators from assuming any JWT can fetch binary evidence; align `docs/api`, app-level OpenAPI description, and route docstrings.

**Output:** Updated `main.py` description and README section; regression test remains green for non-admin **403** on image GET.

## must_haves

1. **`apps/api/src/main.py`** `FastAPI(..., description=...)` (or equivalent) explicitly distinguishes: **ingest** = device key; **list logs** = JWT; **download image** = admin JWT.
2. **`docs/api/README.md`** section for access logs states the same matrix (table or bullet list).
3. **`test_get_access_log_image_requires_admin`** (or successor) still asserts **403** for non-admin and **200** for admin when file exists.

## verification

- Run: `pytest tests/integration/test_endpoints_access_logs.py::TestAccessLogsEndpoints::test_get_access_log_image_requires_admin -q --tb=short`
- Grep: `get_current_admin_user` in `access_logs.py` for image route
- Grep: `admin` and `access_logs` or `log` in `main.py` description string
