## objective

Implement **GATE-01**: `POST /api/v1/gate_control/trigger` must **either** call a configurable downstream actuator and succeed only on verifiable HTTP **2xx**, **or** return an **explicit simulated** payload when no actuator is configured â€” never imply the physical relay fired when nothing ran.

**Purpose:** Replace the current always-`success` stub with honest contracts (CONTEXT **D-01â€“D-04**, **D-08** partial).

**Output:** `Settings` + env vars, `GateTriggerResponse` schema, controller HTTP client (stdlib preferred), updated tests, README + CONCERNS gate bullets.

## must_haves

1. **`GATE_ACTUATOR_URL`** (optional) and **`GATE_ACTUATOR_TIMEOUT_SECONDS`** (default **5**) on `Settings` / `config.py`, documented in **`env.local.example`**.
2. Response JSON includes canonical field **`integration`** with values **`simulated`** or **`live`** (or locked equivalent documented in schema).
3. **Simulated path:** empty/unset `GATE_ACTUATOR_URL` â†’ **200** + `integration: "simulated"` + message that does not claim hardware fired.
4. **Live path:** POST JSON body e.g. `{"action": "open"}` (document exact shape in schema/README); **502** or **503** on timeout, connection error, or non-2xx â€” **fail-fast, no retries** unless you document one retry in code comment.
5. **Admin-only** and auth tests unchanged in spirit (403/401).
