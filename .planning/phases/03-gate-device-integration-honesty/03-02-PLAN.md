## objective

Implement **DEV-01**: device HTTP endpoints are **either** clearly **demonstration** (typed in OpenAPI/responses) **or** **disabled** so production operators cannot treat Bluetooth mocks as live server-side hardware (CONTEXT **D-05–D-07**, **D-08–D-09**).

**Purpose:** Feature-flag + schema honesty; align `main.py` and README with Web Bluetooth reality.

**Output:** `IOT_DEVICE_DEMO_API` setting, dependency guard on routes, `demo: true` on mock responses, tests, docs, CONCERNS device bullet.

## must_haves

1. **`IOT_DEVICE_DEMO_API`:** default **`false`** when `ENVIRONMENT` is `production`/`prod`, else **`true`** unless env explicitly overrides (document in `env.local.example`).
2. When **false**, all **`/api/v1/devices/*`** routes respond with **501** and a **`detail`** stating server-side Bluetooth is disabled / use Web Bluetooth (Portuguese ok).
3. When **true**, extend response models so every **200** response body includes **`demo: true`** (field name locked in OpenAPI).
4. **`main.py`** description + OpenAPI tag description for **devices** as non-production demonstration.
5. **`docs/api/README.md`** table or subsection for device API vs gate (may extend 03-01 gate section).
